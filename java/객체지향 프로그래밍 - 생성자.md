## 생성자

생성자란?

- 객체를 생성할 때 `new` 키워드와 함께 호출
- 인스턴스를 초기화하는 코드가 구현 된다.
- 생성자는 클래스 이름과 동일하다.



### 기본 생성자

- 하나의 클래스에는 반드시 하나 이상의 생성자가 존재해야 한다.
- 개발자가 생성자를 구현하지 않으면 컴파일러가 프리 컴파일 단계에서 생성자 코드를 넣어준다 => **기본 생성자**
  - 기본 생성자는 매개 변수가 없고, 구현부(`{}`내부)가 없다
  - 만약 다른 생성자가 있는 경우 기본 생성자는 제공되지 않는다.

```java
// 기본 생성자
Student studentLee = new Student();
```

생성자를 직접 정의해보자.

```java
package classpart;

public class Student {
    public int studentID;
    public String studentName;
    public String address;
    
    // 생성자 정의
    public Student(int id, String name) {
        studentID = id;
        studentName = name;
    }
}
```

- 이렇게 생성자를 직접 정의해주면 **기본 생성자는 사용할 수 없다**.



### 생성자 오버로딩

그렇다면 생성자를 여러개 정의할 수 있을까? 

그렇다. 이름이 같은 생성자가 여러개여도 매개 변수에 따라 호출되는 생성자가 달라진다. 이를 **생성자 오버로딩**이라고 한다. 

```java
public class Student {
    public int studentID;
    public String studentName;
    public String address;
    
    // 생성자 1
    public Student(int id, String name) {
        studentID = id;
        studentName = name;
    }
    
    // 생성자 2
    public Student(String name) {
        studentName = name;
    }
}
```



### private 변수 접근

만약 멤버 변수에 `private` 키워드가 붙어있는 경우 해당 변수는 클래스 내부에서만 참조할 수 있다. 이러한 경우 생성자를 이용하여 변수에 값을 저장하는 것이 가능하다.

```java
public class Student {
    // 정보 은닉된 변수
    private int studentID;
    public String studentName;
    public String address;
    
    // 생성자 1
    public Student(int id, String name) {
        studentID = id;
        studentName = name;
    }
    
    ...
}
```

```java
public class StudentTest {
    public static void main(String[] args) {
        // studentId에 값 초기화
        Student studentHeo = new Student(100, "허예슬"); 
    }
}
```
